### What is protocol?

> 네트워킹에서 프로토콜은 데이터 서식 지정 및 처리를 위한 규칙이다.
> 네트워크 프로토콜은 컴퓨터의 공통 언어와 같다.
> 네트워크 내의 컴퓨터는 아주 상이한 소프트웨어와 하드웨어를 사용한다.
> 그렇지만 프로토콜을 사용하면 서로 통신할 수 있다.

![[Pasted image 20231004162714.png]]

> 표준화된 프로토콜은 컴퓨터가 할 수 있는 공통 언어와 같다.
> 세계의 다른 지역에서 온 두 사람이 서로의 모국어를 이해 하지 못하지만, 공유하는 제 3 언어를 사용하여 소통하는것과 유사하다.
> 한 컴퓨터에서 인터넷 프로토콜(IP)을 사용하고 두 번째 컴퓨터에서도 그 프로토콜을 사용하는 경우 통신할 수 있다
> 그러나 한 컴퓨터에서 IP를 사용하고 다른 컴퓨터에서 이 프로토콜을 모르면 통신할 수 없다.

### OSI 모델 계층?

> [[OSI 모델 |계방형 시스템 상호 연결(OSI) 모델]]은 인터넷 작동 방식을 추상적으로 표현한 것이다.
> 여기에는 7개의 계층이 포함되어 있으며 각 계층은 네트워킹 기능의 다른 범주를 나타냄

![[Pasted image 20231005090406.png]]

>프로토콜은 이러한 네트워킹 기능을 가능하게 한다. 예를 들어, 인터넷 프로토콜(IP)은 데이터 패킷의 출처와 목적지를 표시 하여 데이터 라우팅을 담당한다.
>IP 덕분에 네트워크 간 통신이 가능하다. 따라서 IP는 네트워크 계층(계층3) 프로토콜로 간주된다.

>또 다른 예로 전종 제어 프로토콜(TCP)은 네트워크를 통한 데이터 패킷[^1]의 전송이 원활하게 진행되도록 보장한다
>따라서 TCP는 전송 계층(계층4) 프로토콜로 간주됩니다.

[^1]: 패킷은 데이터의 작은 부분. 네트워크를 통해 전송되는 모든 데이터는 패킷으로 나뉜다.

### 네트워크 계층에서 실행되는 프로토콜은 무엇인가?

> [!info]+ IPsec
> 인터넷 프로토콜 보안(IPsec)은 가상 사설망(VPN)을 통해 암호화 되고 인증된 IP 연결을 설정한다.
> 기술적으로, IPsec은 하나의 프로토콜이 아니라 캡슐화 보안 프로토콜(ESP), 인증 헤더(AH)
> 보안 연결(SA)을 포함하는 프로토콜 모음이다.

> [!info]+ ICMP
> 인터넷 제어 메세지 프로토콜(ICMP)은 오류를 보고하고 상태 업데이트를 제공한다.
> 예를 들어 라우터가 패킷을 전달할 수 없는 경우 이 프로토콜은 ICMP 메세지를 패킷의 출처로 돌려보낸다.

> [!info]+ IGMP
> 인터넷 그룹 관리 프로토콜(IGMP)은 일대다 네트워크 연결을 설정한다.
> IGMP는 멀티캐스팅을 설정하는 데 도움이 되므로 여러 컴퓨터 하나의 IP 주소로 전달되는 데이터 패킷을 수신할 수 있다.

### 인터넷에서 사용되는 다른 프로토콜

> [!important]+ [TCP]
> 앞서 설명한 것처럼 TCP는 안정적인 데이터 전달을 보장하는 전송 계층 프로토콜이다.
> TCP는 IP와 함께 사용하기위한 것이며 두 프로토콜은 종종 TCP/IP로 함께 참조된다.

> [!important]+ HTTP
> 하이퍼텍스트 전송 프로토콜(HTTP)은 대부분의 사용자가 상호 작용하는 인터넷인 월드와이드웹의 기초이다.
> 장치간에 데이터를 전송하는데 사용된다.
>  HTTP는 [[네트워크 앱 | 어플리케이션]] (예 : 브라우저)이 추가 해석 없이 직접 사용할 수 있는 형식으로 데이터를 저장하므로 애플리케이션 계층(layer 7)에 해당한다.
> OSI 모델의 하위 계층은 어플리케이션이 아닌 컴퓨터의 운영 체제에서 처리된다.
